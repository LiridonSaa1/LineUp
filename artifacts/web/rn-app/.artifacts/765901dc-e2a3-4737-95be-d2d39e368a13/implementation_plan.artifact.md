# Fresh Supabase Setup & Categories Table

This plan details how to reset the Supabase database to a clean state, keeping only the `advertisements` table, and introducing a new `categoria` table with initial data.

## User Review Required

> [!CAUTION]
> **Total Data Reset**: All existing user accounts, barbershops, appointments, and history will be **permanently deleted**. Run the SQL script below in your Supabase SQL Editor.

> [!IMPORTANT]
> **Code Changes**: I will remove the automatic seeding of advertisements from the app code to ensure the app doesn't attempt to modify the database schema or default data on every load.

## Proposed Changes

### [Database] SQL Reset Script
Run this in your Supabase SQL Editor:

```sql
-- 1. Clean up existing tables (except advertisements)
DROP TABLE IF EXISTS subscriptions CASCADE;
DROP TABLE IF EXISTS customers CASCADE;
DROP TABLE IF EXISTS appointments CASCADE;
DROP TABLE IF EXISTS user_locations CASCADE;
DROP TABLE IF EXISTS recent_searches CASCADE;
DROP TABLE IF EXISTS barbershops CASCADE;
DROP TABLE IF EXISTS users CASCADE;

-- 2. Create the 'categoria' table
CREATE TABLE categoria (
  id BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  icon TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 3. Seed 'categoria' with homepage categories
INSERT INTO categoria (name, icon) VALUES
('Haircut & Styling', 'Scissors'),
('Hair Coloring', 'Sparkles'),
('Hair Treatment', 'Zap'),
('Beard & Grooming', 'User'),
('Nails', 'Hand'),
('Makeup', 'Smile'),
('Brows & Lashes', 'Eye'),
('Skin Care', 'Shield'),
('Body Care', 'Waves')
ON CONFLICT (name) DO NOTHING;
```
